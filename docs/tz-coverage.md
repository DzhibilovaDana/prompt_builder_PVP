## 5) Что закрыть до «полного соответствия ТЗ»

1. Добавить формальный документ «Глоссарий + роли + сценарии».
2. Ввести явную API versioning-политику (`/api/v1`) и таблицу DTO/errors.
3. Оформить миграции как отдельный процесс (а не только init-скрипт) + backup/restore регламент.
4. Добавить e2e + a11y проверки (например, Playwright + axe).
5. Подготовить эксплуатационные и пользовательские инструкции в формате, принятом для сдачи по ГОСТ 34.003-90.
